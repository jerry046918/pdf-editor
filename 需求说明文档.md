# PDF桌面编辑器 - 需求说明文档

## 1. 项目概述

### 1.1 项目背景
开发一款面向Windows平台的PDF桌面应用程序，提供常用的PDF操作功能，包括合并、分拆、图片转换和文字编辑等核心功能。

### 1.2 目标用户
- 办公人员：需要处理文档合并、分拆等日常操作
- 学生：整理学习资料、合并课件
- 设计师：将设计图片转换为PDF格式
- 普通用户：简单的PDF编辑需求

### 1.3 核心价值
- **简单易用**：直观的用户界面，无需专业技能
- **功能实用**：覆盖最常用的PDF操作场景
- **本地处理**：无需联网，保护隐私安全
- **性能优良**：快速处理大文件

## 2. 功能需求

### 2.1 PDF合并

#### 功能描述
将多个PDF文件合并为一个PDF文件

#### 用户场景
- 用户需要将多个章节的PDF文档合并为完整文档
- 将多份报告合并为一份完整报告

#### 功能要点
- **文件选择**：支持选择多个PDF文件 ✅
- **顺序调整**：支持上移/下移按钮调整文件合并顺序 ✅
- **预览功能**：显示每个PDF文件的缩略图 ⚠️ (后端已实现，前端需调试)
- **合并选项**：整个文件合并 ✅
- **输出设置**：自定义输出文件名和路径 ✅
- **进度显示**：显示处理状态 ✅

#### 验收标准
| 标准 | 状态 | 说明 |
|------|------|------|
| 支持至少合并20个PDF文件 | ✅ 通过 | 后端支持，前端已实现 |
| 合并后文件大小不超过源文件总和的110% | ⚠️ 待验证 | PyMuPDF优化保存 |
| 合并后页面顺序与用户设定一致 | ✅ 通过 | 按用户选择顺序合并 |
| 100页以内文件合并时间不超过5秒 | ⚠️ 待验证 | 需性能测试 |

#### 接口标准
```typescript
// Tauri Command
pdf_merge(files: string[], output: string) -> PdfResponse

// Python Handler
pdf.merge(params: { files: string[], output: string, options?: {...} })
```

## 3. 技术架构

### 3.1 技术栈选型

#### 最终方案：Tauri 2.x + Python Sidecar + PyMuPDF

**选型理由**：
- **体积优势**：Tauri使用系统WebView，安装包仅3-5MB
- **性能优势**：Rust后端 + PyMuPDF（基于C的MuPDF），处理速度快
- **功能完整**：PyMuPDF提供完整的PDF操作能力
- **开发效率**：前端可用熟悉的Web技术，Python侧有丰富的PDF库生态

**授权说明**：
- 本项目为开源项目
- PyMuPDF使用AGPL-3.0，需注意授权兼容性

### 3.2 技术栈清单

| 层级 | 技术 | 版本 | 用途 |
|------|------|------|------|
| **前端框架** | Svelte | 5.x | 响应式UI框架 |
| | TypeScript | 5.x | 类型安全 |
| | TailwindCSS | 4.x | 样式框架 |
| **桌面框架** | Tauri | 2.x | 主进程框架 |
| | Rust | Edition 2021 | 后端语言 |
| | tauri-plugin-fs | 2.4.5 | 文件系统操作 |
| | tauri-plugin-shell | 2.3.5 | Sidecar进程管理 |
| | tauri-plugin-dialog | 2.6.0 | 文件对话框 |
| **PDF处理** | Python | 3.11+ | Sidecar运行时 |
| | PyMuPDF | 1.24+ | PDF操作核心库 |
| | Pillow | 11.x | 图片处理 |
| **构建工具** | Vite | 6.x | 前端构建 |
| | pnpm | 9.x | 包管理器 |

### 3.3 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                    前端层 (WebView)                          │
│  ┌─────────────────────────────────────────────────────────┐│
│  │  Svelte + TypeScript + TailwindCSS                      ││
│  │  ┌─────────┬─────────┬──────────┬─────────┐             ││
│  │  │ PDF合并 │ PDF分拆 │ 图片转PDF│ 文字编辑│             ││
│  │  │   ✅    │   ✅    │   ⚠️     │   ⚠️    │             ││
│  │  └─────────┴─────────┴──────────┴─────────┘             ││
│  └─────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────┘
                          ↕ Tauri IPC (invoke)
┌─────────────────────────────────────────────────────────────┐
│                  后端层 (Rust Main Process)                  │
│  ┌─────────────────────────────────────────────────────────┐│
│  │  Tauri Plugins: dialog, fs, shell                       ││
│  │  Commands: pdf_merge, pdf_split, etc.                   ││
│  └─────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────┘
                          ↕ stdio (JSON-RPC 2.0)
┌─────────────────────────────────────────────────────────────┐
│               PDF处理层 (Python Sidecar)                     │
│  ┌─────────────────────────────────────────────────────────┐│
│  │  PyMuPDF 核心处理器                                      ││
│  │  Handlers: merge, split, convert, edit, thumbnails      ││
│  └─────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────┘
```

### 3.4 进程通信协议

#### JSON-RPC请求格式
```json
{
  "jsonrpc": "2.0",
  "method": "pdf.merge",
  "params": {
    "files": ["/path/to/file1.pdf", "/path/to/file2.pdf"],
    "output": "/path/to/output.pdf"
  },
  "id": 1
}
```

#### JSON-RPC响应格式
```json
{
  "jsonrpc": "2.0",
  "result": {
    "success": true,
    "page_count": 25,
    "file_size": 1048576
  },
  "id": 1
}
```
