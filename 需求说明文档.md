# PDF桌面编辑器 - 需求说明文档

## 1. 项目概述

### 1.1 项目背景
开发一款面向Windows平台的PDF桌面应用程序，提供常用的PDF操作功能，包括合并、分拆、图片转换和文字编辑等核心功能。

### 1.2 目标用户
- 办公人员：需要处理文档合并、分拆等日常操作
- 学生：整理学习资料、合并课件
- 设计师：将设计图片转换为PDF格式
- 普通用户：简单的PDF编辑需求

### 1.3 核心价值
- **简单易用**：直观的用户界面，无需专业技能
- **功能实用**：覆盖最常用的PDF操作场景
- **本地处理**：无需联网，保护隐私安全
- **性能优良**：快速处理大文件

---

## 2. 功能需求

### 2.1 PDF合并

#### 功能描述
将多个PDF文件合并为一个PDF文件

#### 用户场景
- 用户需要将多个章节的PDF文档合并为完整文档
- 将多份报告合并为一份完整报告

#### 功能要点
- **文件选择**：支持选择多个PDF文件
- **顺序调整**：支持拖拽调整文件合并顺序
- **预览功能**：显示每个PDF文件的页数和缩略图
- **合并选项**：
  - 整个文件合并
  - 选择部分页面合并
- **输出设置**：自定义输出文件名和路径
- **进度显示**：实时显示合并进度

#### 验收标准
- 支持至少合并20个PDF文件
- 合并后文件大小不超过源文件总和的110%
- 合并后页面顺序与用户设定一致
- 100页以内文件合并时间不超过5秒

---

### 2.2 PDF分拆

#### 功能描述
将一个PDF文件分拆为多个PDF文件

#### 用户场景
- 从大型PDF文档中提取特定章节
- 将合同按页码分拆为独立文件

#### 功能要点
- **分拆模式**：
  - 按页码范围分拆（如：1-5, 6-10, 11-15）
  - 按固定页数分拆（如：每5页一个文件）
  - 提取特定页面（如：只提取第3、7、9页）
  - 按书签分拆（如果PDF包含书签）
- **预览功能**：显示PDF总页数和缩略图预览
- **命名规则**：自定义输出文件名前缀，自动添加序号
- **批量输出**：一键输出所有分拆后的文件

#### 验收标准
- 支持至少1000页的PDF文件分拆
- 分拆后每个文件可独立打开
- 页面内容完整无缺失
- 分拆速度：10页/秒

---

### 2.3 多张图片合并转换为PDF

#### 功能描述
将多张图片合并并转换为一个PDF文件

#### 用户场景
- 将扫描的纸质文档图片转为PDF
- 将设计稿、照片整理为PDF文档
- 制作电子相册

#### 功能要点
- **图片格式支持**：
  - 常见格式：JPG, JPEG, PNG, BMP, TIFF, GIF
  - 特殊格式：WebP, HEIC（如果技术可行）
- **图片选择**：
  - 支持选择多张图片
  - 支持文件夹批量导入
  - 支持拖拽导入
- **顺序调整**：
  - 拖拽调整图片顺序
  - 按文件名排序
  - 按时间排序
- **图片处理**：
  - 自动旋转（根据EXIF信息）
  - 图片压缩选项（质量控制）
  - 调整图片尺寸（适应页面）
  - 支持裁剪
- **PDF设置**：
  - 页面大小：A4, Letter, 自定义尺寸
  - 页面方向：自动、横向、纵向
  - 边距设置
  - 一页一张或多张图片（如：2x2布局）
  - 背景色设置（白色/透明）
- **预览功能**：显示PDF效果预览
- **OCR识别**（可选）：提取图片中的文字

#### 验收标准
- 支持至少50张图片合并
- 输出PDF文件大小合理（单张图片不超过2MB时，50张图片PDF不超过100MB）
- 图片质量可接受（用户可调节）
- 转换速度：5张图片/秒（标准质量）

## 3. 非功能需求

### 3.1 性能需求
- **启动时间**：应用启动时间不超过2秒
- **文件加载**：100页PDF加载时间不超过2秒
- **响应时间**：界面操作响应时间不超过200ms
- **内存占用**：空闲状态内存占用不超过60MB，处理大文件时不超过500MB
- **CPU占用**：空闲状态CPU占用不超过5%
- **安装包体积**：不超过25MB

### 3.2 兼容性需求
- **操作系统**：Windows 10, Windows 11
- **架构支持**：x64, ARM64（可选）
- **PDF版本**：支持PDF 1.4到PDF 2.0

### 3.3 可用性需求
- **学习曲线**：新用户10分钟内掌握基本操作
- **界面语言**：简体中文（后续可扩展英文）
- **界面风格**：现代扁平化设计，符合Windows设计规范
- **快捷键**：支持常用快捷键（Ctrl+O打开, Ctrl+S保存等）

### 3.4 可靠性需求
- **稳定性**：连续运行24小时无崩溃
- **异常处理**：优雅处理异常，不丢失用户数据
- **自动保存**：编辑时自动保存临时文件
- **错误恢复**：异常退出后可恢复未保存的工作

### 3.5 安全性需求
- **数据安全**：本地处理，不上传用户文件到服务器
- **临时文件**：安全删除临时文件
- **权限管理**：仅申请必要的文件系统权限

### 3.6 可维护性需求
- **模块化设计**：功能模块独立，易于维护和扩展
- **日志系统**：记录操作日志，便于问题排查
- **版本更新**：支持在线更新功能（可选）

## 4. 技术架构

### 4.1 技术栈选型

#### 最终方案：Tauri + Python Sidecar + PyMuPDF

**选型理由**：
- **体积优势**：Tauri使用系统WebView，安装包仅3-5MB（对比Electron 50-100MB）
- **性能优势**：Rust后端 + PyMuPDF（基于C的MuPDF），处理速度快
- **功能完整**：PyMuPDF提供完整的PDF操作能力，包括合并、分拆、编辑、渲染
- **开发效率**：前端可用熟悉的Web技术，Python侧有丰富的PDF库生态

**授权说明**：
- 本项目为开源项目，使用AGPL-3.0授权
- PyMuPDF同样使用AGPL-3.0，授权兼容

### 4.6 核心API设计

#### 4.6.1 PDF合并 (pdf.merge)

**请求参数**：
```typescript
interface MergeParams {
  files: string[];           // PDF文件路径列表
  output: string;            // 输出路径
  options?: {
    pages?: number[][];      // 每个文件的页面范围，如 [[1,5], [3,10]]
    preserve_bookmarks?: boolean;
  };
}
```

**PyMuPDF实现**：
```python
def merge_pdfs(params: MergeParams) -> dict:
    result = fitz.open()
    for i, file_path in enumerate(params.files):
        src = fitz.open(file_path)
        pages = params.options.pages[i] if params.options.pages else None
        result.insert_pdf(src, from_page=pages[0]-1, to_page=pages[1]-1)
        src.close()
    result.save(params.output)
    return {"success": True, "page_count": len(result)}
```

#### 4.6.2 PDF分拆 (pdf.split)

**请求参数**：
```typescript
interface SplitParams {
  file: string;
  output_dir: string;
  mode: 'range' | 'fixed' | 'extract' | 'bookmark';
  options: {
    ranges?: [number, number][];    // 按范围：[[1,5], [6,10]]
    pages_per_file?: number;        // 固定页数
    pages?: number[];               // 提取特定页
    prefix?: string;                // 输出文件名前缀
  };
}
```

#### 4.6.3 图片转PDF (pdf.convert_images)

**请求参数**：
```typescript
interface ConvertImagesParams {
  images: string[];         // 图片路径列表
  output: string;
  options?: {
    page_size?: 'A4' | 'Letter' | [number, number];  // 页面尺寸
    orientation?: 'auto' | 'portrait' | 'landscape';
    margin?: number;        // 边距（像素）
    quality?: number;       // 图片质量 1-100
    fit?: 'contain' | 'cover' | 'stretch';
  };
}
```

#### 4.6.4 PDF文字编辑 (pdf.edit_text)

**请求参数**：
```typescript
interface EditTextParams {
  file: string;
  output: string;
  operations: Array<{
    type: 'replace' | 'add' | 'delete';
    page: number;
    // 替换
    search?: string;
    replace?: string;
    // 添加
    text?: string;
    position?: [number, number];  // x, y坐标
    font?: {
      family: string;
      size: number;
      color: string;  // hex color
    };
  }>;
}
```

#### 第一阶段：基础框架（2-3周）
- 搭建项目框架
- 实现主界面布局
- 集成PDF处理库
- 实现PDF预览功能

#### 第二阶段：核心功能（4-5周）
- 实现PDF合并功能
- 实现PDF分拆功能
- 实现图片转PDF功能
- 实现PDF文字编辑功能

#### 第三阶段：优化完善（2-3周）
- 性能优化
- 用户体验优化
- 异常处理完善
- 单元测试和集成测试

#### 第四阶段：发布准备（1-2周）
- 安装包制作
- 用户手册编写
- Beta测试
- Bug修复

### 5.2 里程碑

| 里程碑 | 时间节点 | 交付物 |
|--------|---------|--------|
| M1：框架完成 | 第3周 | 可运行的应用框架 |
| M2：核心功能完成 | 第8周 | 所有核心功能可用 |
| M3：测试完成 | 第11周 | 通过所有测试 |
| M4：正式发布 | 第13周 | 安装包和文档 |

## 7. 验收标准

### 7.1 功能验收
- ✅ 所有核心功能（合并、分拆、图片转PDF、文字编辑）正常工作
- ✅ 支持的文件格式正确打开和处理
- ✅ 输出文件格式正确，可在其他PDF阅读器中打开

### 7.2 性能验收
- ✅ 启动时间 ≤ 2秒
- ✅ 100页PDF加载时间 ≤ 2秒
- ✅ 内存占用空闲状态 ≤ 60MB
- ✅ 安装包体积 ≤ 25MB

### 7.3 稳定性验收
- ✅ 连续运行24小时无崩溃
- ✅ 异常情况处理正确
- ✅ 无数据丢失

### 7.4 用户体验验收
- ✅ 界面美观，操作直观
- ✅ 新用户10分钟内掌握基本操作
- ✅ 无明显的界面卡顿
